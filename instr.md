## Подробная инструкция по развертыванию marimo script на GitHub Pages без шаблонного репозитория

### 1. Подготовка маримо-скрипта и данных

- Убедитесь, что у вас есть marimo-скрипт (.py), который вы хотите развернуть.
- Если у вас есть данные (например, CSV файлы или изображения), положите их в отдельную папку, например, `public/`.
- В скрипте обращайтесь к данным так, чтобы пути были относительными к папке с сайтом, например:

```python
import polars as pl
data = pl.read_csv(mo.notebook_location() / "public" / "data.csv")
```

- Для изображений в HTML используйте ссылку `<img src="public/image.png">`.


### 2. Экспорт marimo в статический сайт с WebAssembly

- В вашем локальном окружении с установленным marimo выполните в терминале команду:

```bash
marimo export html-wasm path/to/your_script.py -o docs/ --mode run
```

- Эта команда создаст папку `docs/` с готовым сайтом для размещения на GitHub Pages. Режим `run` создаёт сайт, где интерактивность работает, но редактирование скрипта пользователями выключено. Можно использовать `--mode edit` для возможности редактирования скрипта на сайте.


### 3. Создание GitHub репозитория и загрузка файлов

- Создайте новый публичный репозиторий на GitHub.
- В корне репозитория загрузите содержимое папки `docs/` (полученное на шаге 2).
- Добавьте папку `public/` с данными в корень репозитория рядом с `docs/` (если она не была автоматически скопирована в `docs/public` при экспорте).
- В корне репозитория создайте пустой файл `.nojekyll` (чтобы GitHub Pages не игнорировали файлы и папки, начинающиеся с подчёркивания).


### 4. Настройка GitHub Pages

- В настройках репозитория (Settings > Pages) настройте Source (Источник):
    - Выберите ветку `main` (или другую вашу основную ветку).
    - Выберите папку `/docs` как корень для GitHub Pages.
- Сохраните изменения.


### 5. Проверка и запуск сайта

- Через несколько минут ваш сайт будет доступен по адресу:

```
https://<ваше_имя_пользователя>.github.io/<имя_репозитория>/
```

- Проверьте работу интерактивности — слайдеры, графики, работа с данными должны корректно работать в браузере.
- Для локального тестирования можно запустить:

```bash
python -m http.server -d docs
```

и открыть http://localhost:8000

***

Эта инструкция позволяет вручную разместить marimo-скрипт и связанные данные на GitHub Pages, обеспечив интерактивность через WebAssembly без использования готовых шаблонов и автоматизации в GitHub Actions. Интерактивность обеспечивается runtime Pyodide в браузере, с возможностью загрузки локальных данных из папки `public/`.[^1][^2][^3]
